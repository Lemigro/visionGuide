Projeto Inclusão · Deficiência Visual

VisionGuide
Óculos Assistivo

Protótipo de sistema de visão computacional que atua como "olho digital" para pessoas com deficiência visual — simulado via câmera de PC ou celular.

Março 2026 Fase: Escopo & Prototipação Público: Deficientes Visuais
01 · Visão Geral

O que é o VisionGuide?

Um sistema de IA que usa a câmera do dispositivo para capturar o ambiente em tempo real, processar as imagens com modelos de visão computacional e narrar via áudio tudo que está ao redor do usuário — pessoas, objetos, textos, obstáculos, distâncias e dados contextuais.
02 · Fases do Produto

Roadmap de Desenvolvimento

FASE 01
Protótipo de Simulação (Web/Mobile)
MVP
  • Acesso à câmera via browser (PC ou celular)
  • Captura de frame a cada X segundos
  • Envio da imagem para API de visão (Claude / GPT-4o)
  • Descrição do ambiente em áudio (TTS)
  • Detecção de texto (OCR em tempo real)
  • Identificação de objetos próximos
  • Leitura de produtos (código de barras / rótulo)
  • Modo contínuo e modo sob demanda (botão/voz)
FASE 02
Funcionalidades Avançadas
v2.0
  • Reconhecimento facial (rostos cadastrados)
  • Detecção de obstáculos com alerta sonoro
  • Leitura de cédulas de dinheiro
  • Identificação de cores de roupas/objetos
  • Leitura de cardápios e placas de rua
  • Contexto de localização (GPS + descrição do entorno)
  • Ativação por comando de voz
  • Histórico de leituras recentes
FASE 03
Hardware Físico (Óculos)
v3.0
  • Câmera embarcada (Raspberry Pi / ESP32-CAM)
  • Áudio via bone conduction (sem bloquear ouvido)
  • Processamento local ou offload para nuvem
  • Alimentação por bateria compacta
  • Sensor ultrassônico para distância de obstáculos
  • Vibração para alertas tácteis
  • App mobile como painel de controle
  • Atualização OTA de firmware
TRANSVERSAL
Infraestrutura & Acessibilidade
Infra
  • API de visão (Anthropic Claude / OpenAI GPT-4o)
  • TTS acessível (Google TTS / ElevenLabs)
  • Suporte offline para funções críticas
  • Baixa latência (<2s do clique ao áudio)
  • Interface completamente acessível por voz
  • Logs para avaliação com usuários reais
  • LGPD: dados de imagem não são armazenados
  • Documentação para replicação open-source
03 · Tecnologias

Stack do Protótipo (Fase 1)

Frontend
React + TailwindCSS — interface web acessível via PC e mobile
Câmera API
WebRTC / getUserMedia — acesso à câmera do dispositivo via browser
Visão IA
Claude Vision ou GPT-4o para análise de imagem em tempo real
Text-to-Speech
Web Speech API (nativa) + ElevenLabs para voz natural em PT-BR
Reconhecimento de Voz
Web Speech Recognition API para ativação por comandos vocais
Backend
Node.js / FastAPI — proxy para APIs de IA e gerenciamento de estado
Geolocalização
Geolocation API + Google Maps para contexto do entorno
Privacidade
Frames processados em memória — nenhuma imagem salva em servidor
04 · Diferenciais

Por que o VisionGuide é diferente?

Português nativo
Respostas em PT-BR com contexto brasileiro: real, ruas, produtos nacionais — diferente das soluções israelenses que têm suporte limitado ao idioma.
Custo acessível
OrCam custa R$ 14.900 por unidade. O VisionGuide visa ser open-source e com custo de hardware abaixo de R$ 500.
IA conversacional
O usuário pode fazer perguntas abertas sobre o que está vendo ("o que tem na minha frente?", "quanto custa isso?") em linguagem natural.
Zero instalação
O protótipo roda 100% no browser — sem app, sem configuração. Qualquer smartphone vira o dispositivo de simulação.
Funciona offline
Funções críticas (detecção de obstáculos, OCR básico) devem operar sem internet, garantindo segurança mesmo sem conexão.
Co-design
Desenvolvimento feito em parceria com usuários com deficiência visual desde o MVP — não para eles, mas com eles.

06 · Próximos Passos

O que construir primeiro?

Recomendação: comece pelo protótipo web de captura + descrição de cena. Um app que abre a câmera, captura um frame quando o usuário toca a tela (ou diz "descreva"), envia para a API de visão e fala o resultado em voz alta. Em 1-2 dias é possível ter algo funcional para validar o conceito com usuários reais.
